Platform: Windows
Build: C++

This shouldn’t need explaining right? It’s the classic arcade game ‘Snake’, built to be played in the Windows console. Eat fruit to try and grow as large as you can without eating your tail, or hitting a boundary. Simple!

September 2016 – November 2016 / 9 weeks

This project marks my first foray into C++ programming. It took a little bit of getting used to, but I think so far it is actually my favourite language. I like the control you have over what you’re creating and to me, the file structures look so much more organised ( I’m a sucker for tidy code architecture ).

This was actually quite a difficult project to build. The game itself was no trouble, as the logic to the gameplay is reasonably simple, however, there are many nuances to working with the Windows Console. Writing from scratch obviously means that you don’t have any of the support for graphics, input, game loops, etc that you would in a general game framework.

I took the long timeframe I had for this project to try my hand at building systems I hadn’t had much experience in before. Using tinyxml2, I created a data-driven format for ‘sprite’ (note, it’s all ascii strings) animations, so I needed to build parsers to handle this data and resource loaders and sprite dictionaries for both distribution and accessing these assets. I also tried to integrate several useful design patterns into this project as well, as I had been interested in utilising them in a real project. The first was the state machine, I wrote a basic FSM for handling the game states making full use of polymorphism and inheritance. The second was the ‘Command’ pattern. Although I didn’t manage to finish building this system, I tried to create a robust input system that takes button contexts from xml files and then depending on the game state, the buttons that need executing are passed on to the correct actors.

It was an interesting project to complete and I’m happy that I got the chance to delve into programming some high-level framework features.
